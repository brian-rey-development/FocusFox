Guidelines

Speed firstThe popup must let users start focus in under five seconds. One task selector, one duration, one primary action.
Timer dominanceDuring active and break states, the timer ring is the visual anchor. Secondary actions must stay visually quiet.
Intentional frictionDestructive actions require confirmation. Data reset uses type-to-confirm. There is no accidental exit path.
Consistent surfacesUse white cards on light surfaces and navy panels for blocking, shortcuts, and persistent focus context.
Project color as identityProject color appears in the sidebar, popup selector, timeline, task groups, and stats charts.
Feedback without interruptionToast messages confirm actions, cap at three visible items, and disappear without blocking the workflow.
